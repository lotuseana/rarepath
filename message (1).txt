The Problem

  - 300 million people worldwide live with a rare disease (defined as affecting <1 in 2,000 people)
  - Average time to correct diagnosis: 5-7 years, often involving 7-8 misdiagnoses
  - ~72% of rare diseases are genetic — meaning early detection can change outcomes dramatically
  - 95% of rare diseases have no approved treatment — but diagnosis still matters for management, family planning, and clinical trial eligibility
  - The brutal irony: rare diseases are collectively not rare — they affect more people than cancer

  Why developing countries are worse off

  - Most rare disease research and diagnostic tools are built for patients in the US/EU with access to genomic sequencing, specialist referrals, and comprehensive EHR systems
  - In low/middle-income countries, a child with a rare metabolic disorder will almost certainly be misdiagnosed as malnourished or developmentally delayed — indefinitely
  - Hospitals in these regions have tiny per-disease datasets — too small for any single hospital to train a useful model alone

  ---
  The Core Innovation: Federated Learning Across Sparse Data

  This is the key technical differentiator. Instead of centralizing patient data (impossible due to privacy laws and hospital distrust), the model trains locally at each hospital and only shares encrypted
   model weights.

  Hospital A (50 patients)    Hospital B (30 patients)    Hospital C (80 patients)
        |                           |                           |
    Local Model                Local Model                Local Model
        |                           |                           |
        └───────────────────────────┴───────────────────────────┘
                                    |
                            Aggregated Global Model
                            (no raw patient data ever leaves)
                                    |
                       Push updated weights back to hospitals

  Each hospital improves the global model without exposing patient records. A hospital with only 12 cases of a specific enzyme deficiency contributes meaningfully when pooled with 15 others globally.

  What the System Actually Does

  Input (what a clinician enters)

  - Patient symptoms (free text or structured checklist)
  - Basic lab values (CBC, metabolic panel — available even in low-resource settings)
  - Physical exam findings (dysmorphic features, organ size, neurological signs)
  - Family history
  - Optionally: photos (for dysmorphology recognition via vision model)

  Processing

  1. Symptom encoder — maps clinical narrative to standardized HPO (Human Phenotype Ontology) terms using NLP
  2. Differential generator — ranks candidate rare diseases by probability given the symptom profile
  3. Confidence scoring — flags when more data (specific labs, genetics) would significantly change the ranking
  4. Next-step recommender — tells the clinician exactly what to test next to narrow the diagnosis, prioritized by cost and local availability

  Output

  - Top 5-10 candidate diagnoses with probability scores
  - Plain-language explanation of why each was flagged
  - Suggested confirmatory tests, ordered by cost
  - Links to clinical management guidelines (cached offline)
  - Option to connect to a specialist network for teleconsultation

  ---
  Tech Stack

  For a Hackathon Build (48hrs)

  Frontend:      React or simple HTML (clinician-facing form)
  Backend:       Python + FastAPI
  NLP:           BioBERT or PubMedBERT (fine-tuned on HPO terms)
                 — HuggingFace has pre-trained versions ready to use
  Federated sim: Flower (flwr) — Python federated learning framework,
                 surprisingly easy to mock with 3 local nodes
  Differential:  Bayesian network over HPO-to-disease graph
                 OR embedding similarity (symptom vector vs disease vectors)
  Database:      SQLite for hackathon, PostgreSQL for real
  Deployment:    Docker + any cloud (or localhost for demo)

  For a Real Product

  - Add FHIR integration (healthcare data standard) so it plugs into existing hospital systems
  - Replace simulated federation with PySyft or TensorFlow Federated for production-grade privacy
  - Add genomic input when sequencing is available (VCF file parsing)
  - Mobile-first design for tablet use in clinics without desktops
